Iranian Payment Gateway
Without a Database

سرویس مستقل پرداخت با پشتیبانی از زیبال، زرین‌پال و سپ — نتایج از طریق وب‌هوک به اپلیکیشن شما ارسال می‌شود.

Standalone payment service supporting Zibal, Zarinpal, and SEP — results delivered to your app via webhooks.

Zibal Zarinpal SEP / Shaparak بدون دیتابیس No Database Fail-over خودکار Auto Fail-over MIT Node.js 18+

این سرویس چیست؟What Is This?

ir-payment-gateway یک میکروسرویس مستقل است که بین اپلیکیشن شما و درگاه‌های بانکی ایران قرار می‌گیرد.

ir-payment-gateway is a standalone microservice sitting between your application and Iranian payment gateways.

دریافت نتیجه (وب‌هوک)Receiving Result (Webhook)

پس از تأیید یا رد، سرویس یک POST به URL‌های تعریف شده ارسال می‌کند:

After verification or failure, the service POSTs to your configured URLs:

✓ PAYMENT_SUCCESS_WEBHOOK_URL

json
{ "orderId":  "order-2024-001",
  "gateway":  "sep",
  "code":     "REF123456",
  "status":   "success",
  "verifyDetails": { ... },
  "timestamp": "2024-01-01T12:00:00Z" }

✗ PAYMENT_FAILURE_WEBHOOK_URL

json
{ "orderId": "order-2024-001",
  "gateway": "sep",
  "code":    "",
  "status":  "failed",
  "reason":  "توسط کاربر انصراف داده شد",
  "timestamp": "2024-01-01T12:00:00Z" }

Handler نمونه در Express.js:

Example Express.js handler:

javascript
app.post('/payments/success', async (req, res) => {
  const secret = req.headers['x-webhook-secret'];
  if (secret !== process.env.WEBHOOK_SECRET)
    return res.status(401).json({ error: 'Unauthorized' });

  const { orderId, code, status } = req.body;
  if (status === 'success') {
    await db.orders.update({ id: orderId },
      { status: 'paid', refCode: code });
  }
  res.json({ ok: true }); // always respond with 200
});

مهم: اگر endpoint شما کد غیر 200 برگرداند، وب‌هوک به صف retry اضافه شده و با backoff نمایی مجدداً ارسال می‌شود.

Important: Non-200 response adds the webhook to the retry queue with exponential back-off.

Fail-over خودکارAutomatic Fail-over

اگر درگاه اصلی timeout بخورد یا خطا بدهد، سرویس به درگاه بعدی سوئیچ می‌کند:

If the primary gateway times out or errors, the service switches to the next in the chain:

SEP (Saman)
زنجیره: SEP → Zarinpal
Chain: SEP → Zarinpal
اگر SEP در ۸ ثانیه پاسخ ندهد، به زرین‌پال fail-over می‌شود
If SEP doesn't respond in 8s, fails over to Zarinpal
Zarinpal
زنجیره: Zarinpal (فقط)
Chain: Zarinpal only
آخرین لینک — timeout طولانی‌تر
Last in chain — longer timeout applies
Zibal
زنجیره: Zibal (فقط)
Chain: Zibal only
بدون fail-over
No fail-over chain
غیرفعال کردن
Disable
PAYMENT_FAILOVER_ENABLED=false
همیشه از درگاه انتخابی استفاده می‌شود
Always uses the selected gateway

  --name payment-gw -p 3000:3000 \

yaml
services:
  payment-gateway:
    build: .
    ports: ["3000:3000"]
    env_file: .env
    volumes:
      - ./data:/app/data
      - ./logs:/app/logs
    restart: unless-stopped
    healthcheck:
      test: ["CMD","wget","-qO-","http://localhost:3000/api/test/health"]
      interval: 30s

پایداری داده: پوشه‌های data/ و logs/ را حتماً mount کنید تا در صورت restart داده‌ها حفظ شوند.

Data persistence: Always mount data/ and logs/ so data survives container restarts.
